<아이디 중복 체크>

1. url 설정 : /check/id

2. 화면 구성(이미 회원가입페이지에 버튼이 추가됨)

3. signup.jsp
 - 아이디 중복체크 클릭 이벤트 등록
   - 이벤트 동작 확인(console.log 나 alert을 이용)
 - 전역 변수 추가 : idCheck
   - 기본값은 false
 - 아이디 중복체크 클릭 이벤트에 기능 구현
   - 입력한 아이디를 가져옴
   - 아이디 예외처리 및 알림
   - ajax 샘플 코드 추가
     - 보내는 데이터를 회원 아이디가 들어있는 member 객체를 생성
     - 서버에서 보낸 값에 따라 아이디 사용 가능/불가능을 알림
     - 사용 가능한 경우 idCheck변수를 true로 설정

4. 컨트롤러
 - 메소드 추가
   - @ResponseBody 추가
   - value : /check/id
   - method : post
   - 리턴 : Map<String, Object>
 - 기능 구현
   - 화면에서 보낸 회원 정보에 있는 아이디 확인(sysout)
   - 서비스에게 회원 정보를 주면서 아이디가 중복됐는지 확인해달라고 요청
   - 확인 결과를 map에 추가

5. 서비스/임플
 - 메소드 추가
 - 기능 구현
   - 회원 null체크, 아이디 null체크, 아이디 유효성검사
   - 다오에게 아이디를 주면서 회원 정보를 가져오라고 요청
   - 가져온 회원정보가 null이면 true를 리턴
   - 있으면 false를 리턴

6. signup.jsp
 - validate에서 전송 되기전 아이디 중복체크 여부를 확인하여 중복체크를 안했으면 회원가입이 진행되지 않게 작업
   - submitHandler 속성을 추가 
 - 아이디 변경 이벤트 추가 
   - 아이디가 변경되면 idCheck값을 false로 수정
